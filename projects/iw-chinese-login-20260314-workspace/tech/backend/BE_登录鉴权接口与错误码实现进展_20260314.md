## 接口定义（联调口径）
### 1) 登录接口
- Method: `POST`
- Path: `/api/v1/auth/login`
- Content-Type: `application/json`

### 请求体
```json
{
  "account": "string, required, 手机号/邮箱/用户名",
  "password": "string, required, minLength=8",
  "rememberMe": "boolean, optional"
}
```

### 成功响应
```json
{
  "code": "AUTH_0000",
  "message": "登录成功",
  "data": {
    "accessToken": "<jwt>",
    "refreshToken": "<opaque_or_jwt>",
    "expiresIn": 7200,
    "user": {
      "id": "u_1001",
      "name": "张三",
      "roles": ["user"]
    }
  },
  "requestId": "trace-id"
}
```

### 失败响应（统一）
```json
{
  "code": "AUTH_1001",
  "message": "账号或密码有误，请重新输入",
  "data": null,
  "requestId": "trace-id"
}
```

## 错误码（首版）
- `AUTH_0000`：登录成功
- `AUTH_1001`：账号或密码错误（统一提示，不暴露账号存在性）
- `AUTH_1002`：账号格式不合法
- `AUTH_1003`：密码格式不合法（长度不足 8 位）
- `AUTH_1004`：账号已被锁定
- `AUTH_1005`：请求过于频繁，请稍后重试
- `AUTH_1099`：系统繁忙，请稍后重试

## 安全与风控约束
1. 失败提示统一使用 `AUTH_1001` 文案，避免用户名枚举。
2. 登录失败阈值触发账号临时锁定（建议 5 次/15 分钟）。
3. 对同账号+IP 组合启用限流与冷却窗口。
4. 响应统一回传 `requestId` 用于日志追踪。
