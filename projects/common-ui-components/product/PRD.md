# 通用基础UI组件需求说明书（PRD）
## 1. 背景
为统一全项目前端交互规范，降低重复开发成本，提取线框图中通用基础组件作为标准化组件库输出，覆盖所有业务线加载、验证、模块化、国际化场景。

## 2. 业务目标
- 通用组件复用率≥90%
- 新页面开发效率提升30%
- 跨端交互体验一致性100%

## 3. 适用范围
所有Web/App/H5端业务项目，优先级：中后台系统 > C端应用 > 活动页

## 4. 功能需求
### 4.1 加载状态组件
| 状态 | 说明 | 交互规则 |
|------|------|----------|
| 默认加载 | 页面/模块初始加载 | 显示Loading动画 + 骨架屏占位，加载超时≥10s自动触发重试提示 |
| 加载错误 | 资源加载失败 | 显示错误提示 + 重试按钮，点击后重新发起请求 |
| 骨架屏 | 内容预加载占位 | 与实际内容布局1:1匹配，支持自定义行数/列数/模块结构 |

### 4.2 验证码组件
| 状态 | 说明 | 交互规则 |
|------|------|----------|
| 正常状态 | 验证码展示 | 支持图片/滑动/短信类型切换，点击刷新按钮可更换验证码 |
| 过期状态 | 验证码超时 | 自动提示"验证码已过期"，默认15分钟有效期，可配置 |
| 错误状态 | 验证失败 | 错误次数≥3次触发滑块验证，≥5次锁定1分钟 |

### 4.3 Lego模块化组件
| 类型 | 说明 | 交互规则 |
|------|------|----------|
| 基础 Lego 块 | 最小可复用模块 | 支持拖拽排序、显隐配置、参数自定义 |
| 组合 Lego 块 | 多模块组合 | 支持保存为自定义模板，一键复用 |
| 全局配置 | 统一管理 | 所有Lego块样式、行为支持全局统一配置修改 |

### 4.4 多语言切换组件
| 功能 | 说明 | 交互规则 |
|------|------|----------|
| 语言切换 | 多语言支持 | 默认支持中/英/日/韩，可扩展，切换后全站内容实时生效 |
| 记忆功能 | 偏好存储 | 自动记录用户上次选择的语言，下次访问自动应用 |
| 兜底规则 | 缺失处理 | 未翻译内容自动展示英文兜底，不显示乱码 |

## 5. 非功能需求
- 组件体积：单组件压缩后≤5KB
- 兼容性：支持Chrome≥90、Safari≥14、微信小程序≥2.0
- 性能：组件渲染时间≤100ms，无卡顿

## 6. 验收标准
1. 所有组件符合设计规范，交互与PRD描述一致
2. 组件在极端场景（网络差、低配置设备）下无崩溃、无异常
3. 提供完整的组件使用文档与Demo示例
4. 单元测试覆盖率≥80%

## 7. 依赖与风险
- 依赖：前端基础框架（Vue3/React）版本统一
- 风险：旧项目升级适配成本，需预留1周适配时间

## 8. 交付物
- 可直接使用的组件库包
- 组件使用文档与Demo站点
- 旧项目升级适配指南
