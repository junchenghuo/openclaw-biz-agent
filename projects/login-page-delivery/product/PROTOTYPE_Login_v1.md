# 登录页原型（v1）

> 用途：给发起人/团队快速对齐“页面长什么样、交互怎么走”。

## 1. 页面信息
- 路由：`/login`
- 登录方式：用户名/账号 + 密码（本期不含注册/找回密码/短信验证码/滑块）
- 成功后跳转：默认建议 `/dashboard`（也可按 `?redirect=` 回跳来源页）

## 2. 线框原型（文字版）

```
┌──────────────────────────────────────────┐
│                 背景/留白                 │
│                                          │
│        ┌──────────────────────────┐       │
│        │   Logo(可选)  产品名(可选) │       │
│        │   标题：欢迎登录           │       │
│        │                          │       │
│        │  [ 用户名/账号输入框 ]     │       │
│        │   - 错误提示（字段下方）   │       │
│        │                          │       │
│        │  [ 密码输入框 ][👁]        │       │
│        │   - 错误提示（字段下方）   │       │
│        │                          │       │
│        │  (Alert/Toast) 全局错误提示 │       │
│        │                          │       │
│        │  [ 登录按钮（loading/disabled）] │
│        └──────────────────────────┘       │
│                                          │
└──────────────────────────────────────────┘
```

## 3. 交互流程
### 3.1 初始
- 用户名/密码为空
- 登录按钮 disabled

### 3.2 校验
- blur 时提示字段级错误
- 点击登录时若不通过：聚焦到第一个错误字段

### 3.3 提交
- 点击登录或在密码框 Enter：提交一次请求
- 提交中：按钮 loading（“登录中…”），禁用输入与按钮，防重复提交

### 3.4 成功
- 成功后保存会话（Cookie/Token 以最终投票/后端实现为准）
- 跳转到：
  - 若存在 `redirect`：跳转到 `redirect`（建议做白名单/同域限制）
  - 否则：`/dashboard`

### 3.5 失败（最小覆盖）
- 账号/密码错误：提示“用户名或密码错误”
- 无权限/禁用：提示“账号无权限或已被禁用，请联系管理员”
- 网络/5xx：提示“服务异常，请稍后重试”

## 4. 验收（摘自 PRD）
- UI 美观、布局稳定（1366x768 / 1920x1080 不溢出）
- 校验与错误提示明确
- loading/禁用/防重复提交齐全
- 成功后跳转正确
