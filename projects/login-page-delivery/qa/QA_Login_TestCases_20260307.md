# 登录页测试用例 / 验收 Checklist（T6）

> 范围：简单登录页（用户名 + 密码），对接后端登录接口，本地部署。

## 1. 基本假设（待确认）
- 表单字段：用户名（或邮箱/手机号）+ 密码；按钮「登录」。
- 接口：`POST /login`（示例），成功返回 token / session；失败返回错误码与文案。
- 登录成功后页面跳转到首页/工作台；并持久化登录态（cookie/localStorage 之一）。
- 统一错误提示：字段级校验错误 + 接口错误提示。

> 如 PRD/接口文档有差异，以实际为准（建议补充接口约定：状态码、错误码、限流/锁定策略）。

## 2. 测试用例（正向 / 反向 / 异常 / 安全基础）

### 2.1 UI/交互与表单校验
| 编号 | 场景 | 步骤 | 期望 |
|---|---|---|---|
| UI-01 | 默认态 | 打开登录页 | 两个输入框与登录按钮展示正常；初始无报错 |
| UI-02 | Tab 顺序 | 光标从用户名开始，Tab 切换 | 顺序：用户名 -> 密码 -> 登录按钮；可回退 |
| UI-03 | Enter 提交 | 输入合法用户名/密码，按 Enter | 等价于点击登录触发提交 |
| UI-04 | 密码掩码 | 输入密码 | 默认掩码显示；若有“显示/隐藏”按钮则切换正常 |
| UI-05 | 登录按钮禁用策略 | 用户名/密码为空 | 按钮禁用或点击后提示必填（与设计一致） |
| UI-06 | 用户名空校验 | 用户名为空、密码随意，点击登录 | 提示“用户名必填”（文案按设计） |
| UI-07 | 密码空校验 | 用户名随意、密码为空，点击登录 | 提示“密码必填” |
| UI-08 | 用户名长度边界 | 输入最小/最大长度、超出最大长度 | 最大长度限制或给出提示；不应崩溃 |
| UI-09 | 密码长度边界 | 输入最小/最大长度、超出最大长度 | 规则符合 PRD；不应崩溃 |
| UI-10 | 去空格策略 | 用户名前后空格、密码前后空格 | 与产品约定一致（常见：用户名 trim，密码不 trim）；提示明确 |
| UI-11 | 多次点击/防抖 | 连续快速点击登录 | 只发起一次请求或后续点击无效；无重复提交 |
| UI-12 | 加载态 | 点击登录后接口未返回 | 按钮 loading/禁用；出现“登录中…”等状态；返回后恢复 |
| UI-13 | 错误提示复位 | 一次失败后修改输入重新提交 | 错误提示按规则更新/消失；不残留旧错误 |

### 2.2 正向（成功登录）
| 编号 | 场景 | 步骤 | 期望 |
|---|---|---|---|
| POS-01 | 使用正确账号密码登录 | 输入正确用户名+密码，提交 | 接口返回成功；跳转目标页；登录态保存 |
| POS-02 | 刷新保持登录态 | 成功登录后刷新页面 | 仍保持登录（若要求）；不会回到登录页 |
| POS-03 | 已登录访问登录页 | 登录后手动访问 `/login` | 自动跳转到目标页（若要求） |

### 2.3 反向（业务失败）
| 编号 | 场景 | 步骤 | 期望 |
|---|---|---|---|
| NEG-01 | 用户不存在 | 输入不存在用户名，任意密码提交 | 明确错误提示（不泄露是否存在用户：按安全要求可统一提示“账号或密码错误”） |
| NEG-02 | 密码错误 | 正确用户名 + 错误密码提交 | 提示错误；不登录；保持在登录页 |
| NEG-03 | 账号被禁用/冻结（如有） | 使用禁用账号提交 | 提示账号状态；不登录 |
| NEG-04 | 必填校验拦截 | 用户名或密码为空提交 | 不发送接口或接口返回前端直接提示（与实现一致） |

### 2.4 异常/网络/系统错误
| 编号 | 场景 | 步骤 | 期望 |
|---|---|---|---|
| ERR-01 | 接口 500 | mock /login 返回 500 | 友好提示“服务异常，请稍后重试”；可再次提交 |
| ERR-02 | 超时 | mock 超时 | 提示超时；loading 结束；可重试 |
| ERR-03 | 断网 | 断网后点击登录 | 提示网络不可用；不崩溃 |
| ERR-04 | 返回数据缺字段 | 接口返回缺 token 等字段 | 前端容错：提示异常；不进入登录态 |
| ERR-05 | 频率限制 429（如有） | 多次失败触发限流 | 提示稍后再试；符合后端策略 |

### 2.5 安全基础项（前端验收维度）
| 编号 | 场景 | 检查点 | 期望 |
|---|---|---|---|
| SEC-01 | HTTPS/本地安全 | 生产环境是否强制 HTTPS（本地可 HTTP） | 生产不明文传输；本地按环境说明 |
| SEC-02 | 密码不落日志 | 检查前端日志/埋点/错误上报 | 不记录明文密码 |
| SEC-03 | 错误信息不泄露 | 用户不存在/密码错误提示 | 统一/不暴露用户存在性（推荐） |
| SEC-04 | XSS 注入 | 用户名输入 `<script>` 等提交 | 页面不执行脚本；输出/提示做转义 |
| SEC-05 | Token 存储策略 | 登录成功后的 token 存储 | 与安全要求一致（优先 httpOnly cookie；若 localStorage 需评估 XSS 风险） |
| SEC-06 | CSRF（若 cookie） | cookie session 场景 | 有 CSRF 防护（SameSite/Token） |

## 3. 验收 Checklist（可作为上线门槛）

### 功能
- [ ] 表单校验：必填、长度/格式规则与 PRD 一致
- [ ] 登录成功：正确跳转到目标页面
- [ ] 登录失败：错误提示清晰；不会误进入登录态
- [ ] 防重复提交：loading/禁用状态正常

### 接口与兼容
- [ ] 异常处理：500/超时/断网均有兜底提示且可重试
- [ ] 返回字段缺失/异常时不崩溃
- [ ] 主流浏览器（Chrome/Safari/Edge）基本可用（如需移动端再补）

### 安全基础
- [ ] 不记录明文密码到日志/埋点
- [ ] 错误信息不泄露敏感细节（推荐统一“账号或密码错误”）
- [ ] Token/Session 存储策略符合团队约定（如 httpOnly cookie 优先）

### 可用性
- [ ] 文案可理解；输入框可复制/粘贴；键盘操作（Tab/Enter）可用
- [ ] 视觉状态：错误态、focus 态、disabled/loading 态清晰
