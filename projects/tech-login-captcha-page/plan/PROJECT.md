# 项目：科技风登录页（含图形验证码）

## 0. 需求确认（Gate-0）
- 发起人：霍钧城
- 目标：交付一个具有科技风格的登录页面，并在登录时增加图形验证码。
- 期望交付：
  - 前端登录页（含用户名/密码/验证码输入、验证码图片刷新、提交/错误提示）
  - 可接入后端登录接口（如无现成接口，提供 Mock 方案）
  - 适配：PC 优先（需确认是否移动端）
  - 交付物：代码仓库/页面地址、使用说明、截图

## 1. 待确认问题（需要老板拍板）
1) 目标形态：Web H5/管理后台/桌面端？是否需要响应式？
2) 技术栈：现有项目是 Vue3/React/原生？是否有现成 UI 组件库（AntD/ElementPlus）？
3) 验证码来源：
   - A. 后端生成（推荐：/captcha 接口返回 base64 + captcha_id）
   - B. 第三方服务
   - C. 前端生成（安全性较弱，一般不建议）
4) 登录接口：是否已有 /login ？字段是 username/password 还是 phone/code？
5) 视觉要求：是否有品牌色/Logo/背景图资源？“科技风”偏霓虹/玻璃拟态/深色赛博？
6) 文案与错误提示：需要中英文？
7) 交付时间：期望上线/验收时间点？

## 2. 验收标准（草案）
- 页面视觉：符合“科技风”设计；布局整洁；加载无明显抖动。
- 验证码：
  - 默认展示验证码图片；点击可刷新；输入为空/错误有提示；登录失败时自动刷新一次。
- 交互：Enter 提交；按钮 loading；错误提示可读。
- 兼容：Chrome/Edge 最新版。

## 3. 里程碑（草案）
- D0：需求确认（Gate-0）
- D1：PRD+原型评审（Gate-1）
- D2：UI 视觉稿 + 架构/接口约定评审（Gate-2）
- D3-D4：开发实现（Gate-3）
- D5：测试放行（Gate-4）
- D6：部署发布（Gate-5）
