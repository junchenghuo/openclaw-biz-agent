# PRD: 简单账号密码登录（MVP）

- 任务ID: TASK-001
- Owner: PM
- 日期: 2026-03-05
- 版本: v1.0

## 1. 背景与目标
为当前项目提供最小可用登录能力，仅支持“账号 + 密码”方式，满足基础登录与会话建立，不包含扩展认证能力。

## 2. 范围
### 2.1 In Scope（本期包含）
1. 登录页输入项：账号、密码。
2. 前端基础校验：
   - 账号不能为空。
   - 密码不能为空。
3. 登录提交与结果反馈：
   - 成功：跳转到系统首页（或占位首页）。
   - 失败：显示统一错误提示（账号或密码错误）。
4. 基础安全要求：
   - 密码输入框默认掩码显示。
   - 传输层要求 HTTPS（环境具备时必须开启）。

### 2.2 Out of Scope（本期不做）
1. 短信验证码登录。
2. 第三方登录（微信/Google/Apple 等）。
3. 忘记密码/重置密码流程。
4. 注册流程。
5. 图形验证码、风控策略、设备管理、多因素认证。

## 3. 用户故事
- 作为用户，我希望输入账号和密码后可以登录系统。
- 作为用户，我在输入错误时能得到明确且统一的失败提示。

## 4. 交互与规则
1. 点击“登录”按钮时触发校验与提交。
2. 当账号或密码为空时，不发起请求，直接提示“请输入账号/密码”。
3. 服务端返回认证失败时，提示“账号或密码错误”。
4. 登录成功后进入首页并建立登录态（token/session 具体由架构设计定义）。

## 5. 非功能要求
1. 可用性：
   - 页面在主流桌面浏览器可正常使用（Chrome/Safari 最新稳定版）。
2. 性能：
   - 在正常网络下，登录接口响应目标 < 1s（P50，非强约束，作为体验目标）。
3. 安全：
   - 不在前端明文持久化密码。
   - 错误提示不区分“账号不存在”与“密码错误”。

## 6. 验收标准（Acceptance Criteria）
### AC-01 页面与输入
- 打开登录页可看到账号、密码输入框及登录按钮。

### AC-02 空值校验
- 账号为空点击登录，提示“请输入账号”，且不请求后端。
- 密码为空点击登录，提示“请输入密码”，且不请求后端。

### AC-03 登录成功
- 输入有效账号密码，返回成功后跳转首页并进入登录态。

### AC-04 登录失败
- 输入错误账号或密码，提示“账号或密码错误”，停留在登录页。

### AC-05 安全与交互
- 密码框默认掩码。
- 错误提示不暴露账号存在性。
