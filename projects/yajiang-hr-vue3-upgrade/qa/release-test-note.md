# 雅江 HR Vue3 升级发布测试说明（草案）

## 1. 测试目标
- 验证 Vue2 -> Vue3 升级后核心业务功能等价。
- 验证性能不劣化、日志可观测、发布可回滚。

## 2. 测试范围
- 登录、首页、员工信息管理、组织架构、审批列表、入转调离核心链路。
- 浏览器：Chrome / Edge。

## 3. 当前阻塞与解除条件
- 阻塞1：可测环境信息未齐（地址、账号、数据准备）。
- 阻塞2：日志口径未统一（字段、traceId、错误码、采集位置）。

解除条件：
1) BE 在 `tech/backend/BE_COMPAT_YAJIANG_HR_VUE3.md` 补齐环境与日志约定。
2) OPS 在 `ops/DEPLOY_YAJIANG_HR_VUE3.md` 补齐环境开通与观测配置。

## 4. 执行策略
- 第一轮：冒烟（20 分钟内）
- 第二轮：核心流程回归
- 第三轮：非功能（性能、兼容性、可观测性）
- 缺陷闭环：按 P1/P2/P3 分级，P1 必须清零

## 5. 产出物
- `qa/smoke-checklist.md`
- `qa/regression-checklist.md`
- `qa/TEST_REPORT_YAJIANG_HR_VUE3.md`（执行后）

## 6. 放行口径（QA）
- 放行：P1=0，核心流程 100% 通过，日志链路可追。
- 有条件放行：存在可控 P2，且有明确修复与监控方案。
- 不放行：存在 P1 或核心链路失败或日志不可追。
