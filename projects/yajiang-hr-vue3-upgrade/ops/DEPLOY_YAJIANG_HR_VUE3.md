## 2. 观测口径（日志/监控/告警）

### 2.1 日志字段基线（最小可用集）
> 与 BE 对齐后定版。先按统一字段占位。

| 字段 | 说明 | 是否必填 |
|---|---|---|
| `timestamp` | 事件时间（ISO8601） | 是 |
| `level` | 日志级别（INFO/WARN/ERROR） | 是 |
| `traceId` | 全链路追踪 ID | 是 |
| `requestId` | 请求 ID（网关/服务） | 建议 |
| `userId` | 用户标识（脱敏） | 建议 |
| `endpoint` | 接口路径 | 是 |
| `method` | HTTP 方法 | 是 |
| `statusCode` | HTTP 状态码 | 是 |
| `errorCode` | 业务错误码 | 错误场景必填 |
| `latencyMs` | 接口耗时 | 是 |

### 2.2 采集位置（占位）
- 前端：浏览器错误、关键交互失败（可选上报）
- 网关层：访问日志、4xx/5xx
- 后端服务：应用日志、错误栈、慢请求
- 基础设施：主机/容器资源、网络与磁盘

### 2.3 监控指标与告警阈值（初版）
- 可用性：5 分钟成功率 < 99.5% 告警
- 错误率：5xx 比例 > 1% 告警
- 时延：P95 > 800ms 连续 10 分钟告警
- 资源：CPU > 80%（10 分钟）、内存 > 85%（10 分钟）告警
- 链路：`traceId` 缺失率 > 1% 告警

### 2.4 验收 Gate（QA 执行前）
- [ ] 关键链路日志包含 `traceId`
- [ ] 错误码与接口文档一致
- [ ] 监控看板可见核心指标
- [ ] 告警可触发并可路由到值班通道

## 3. 回滚预案（占位版）

### 3.1 触发条件
- 发布后出现持续 5xx、核心流程不可用、数据一致性风险。

### 3.2 回滚策略
1. 立即冻结变更（停止继续发布）。
2. 切回上一稳定版本（前端静态资源 + 后端服务版本）。
3. 执行回滚后健康检查：
   - 首页与登录可用
   - 核心 API 200/业务错误码正常
   - 关键监控恢复阈值内
4. 在 15 分钟内完成对内通报（现象、影响、处置、下一步）。

### 3.3 回滚时限（SLO）
- 发现故障到完成回滚：目标 ≤ 30 分钟。
