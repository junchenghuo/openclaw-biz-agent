## 1. 背景

用户身份认证是系统的入口功能，承载注册、登录、登出、密码找回等核心场景。当前缺乏统一的登录模块，新用户无法注册、老用户无法安全登录，影响系统可用性。

## 2. 目标

- [ ] 实现用户注册功能，支持用户名/密码方式
- [ ] 实现用户登录功能，校验 credentials
- [ ] 实现登出功能，清除会话
- [ ] 实现密码找回流程（可选 v2）
- [ ] 前后端联调通过，测试覆盖核心路径

## 3. 用户故事

| ID | 角色 | 故事 | 优先级 |
|----|------|------|--------|
| US-1 | 新用户 | 我想注册账号，以便成为系统用户 | P0 |
| US-2 | 已注册用户 | 我想用账号密码登录，以便进入系统 | P0 |
| US-3 | 已登录用户 | 我想点击登出，以便安全退出 | P0 |
| US-4 | 已注册用户 | 我想找回密码，以便恢复账号访问 | P1 |

## 4. 功能范围

### 4.1 注册模块
- 用户名（唯一校验）
- 密码（明文传输 HTTPS，后端 bcrypt 存储）
- 确认密码
- 密码强度要求：≥8位，含大小写+数字

### 4.2 登录模块
- 用户名/密码登录
- 记住我（7天免登录）
- 登录失败计数（5次后锁定15分钟）

### 4.3 登出模块
- 清除 session/token
- 跳转至登录页

### 4.4 密码找回（v2 规划）
- 邮箱验证码校验
- 密码重置

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 登录响应 ≤ 500ms |
| 安全 | 密码 bcrypt 存储，密码字段禁止明文日志 |
| 兼容性 | Chrome / Edge / Safari 最新版 |
| 可用性 | 错误提示明确，表单校验实时反馈 |

## 6. 验收标准

| 场景 | 预期结果 |
|------|----------|
| 注册成功 | 跳转登录页，提示"注册成功，请登录" |
| 用户名重复 | 实时提示"用户名已存在" |
| 密码强度不足 | 实时提示密码要求 |
| 登录成功 | 跳转首页，显示用户名 |
| 登录失败 | 提示"用户名或密码错误" |
| 登录失败5次 | 提示"账号已锁定，请15分钟后重试" |
| 登出成功 | 跳转登录页，清除会话 |

## 7. 依赖

- 后端：用户表设计、密码加密库、Session/Token 机制
- 前端：UI 组件库、表单校验
- 测试：测试账号数据准备

## 8. 风险

- 前后端数据格式需对齐（请求/响应结构）
- 密码找回涉及邮件服务，v1 暂不实现
