# 用途
- 提高覆盖率并可追踪验收。

# 触发关键词
- 测试用例
- 测试矩阵
- 覆盖率
- 验收测试

# 标准流程
1. 提取功能点与风险点。
2. 建立正向/逆向/边界用例矩阵。
3. 标注优先级与执行顺序。
4. 输出缺陷记录模板。

# 输出产物
- test-matrix.md
- testcase-list.md
- defect-template.md

# 约束
- 优先使用本地文件与离线流程，不调用付费 API。
- 不要求任何额外 KEY（如 OPENAI_API_KEY、第三方云厂商 KEY）。
