## 2. 模型选型

### 2.1 验证码识别（OCR）

| 方案 | 模型 | 准确率 | 延迟 | 成本 | 适用场景 |
|------|------|--------|------|------|----------|
| 推荐 | Tesseract OCR (本地) | 85-90% | <200ms | 免费 | 简单验证码 |
| 备选 | EasyOCR | 92-95% | <500ms | GPU | 复杂验证码 |
| 云服务 | AWS Textract | 98%+ | <1s | 按调用 | 高准确率需求 |

**推荐配置**：Tesseract OCR + 自定义预处理（灰度、二值化、去噪）

### 2.2 登录异常检测

| 方案 | 模型 | 场景 | 延迟 | 成本 |
|------|------|------|------|------|
| 推荐 | Qwen-7B-Chat | 实时行为分析 | <1s | API |
| 备选 | 本地规则引擎 | 固定规则匹配 | <50ms | 免费 |

## 3. 提示策略

### 3.1 验证码识别提示

```markdown
你是一个验证码识别专家。请识别图片中的验证码字符。

要求：
1. 只输出识别结果，不包含任何解释
2. 如果识别不确定，输出 "FAIL"
3. 返回格式：{ "result": "ABC123" }

图片：[base64编码]
```

### 3.2 异常登录检测提示

```markdown
分析以下登录日志，识别是否存在异常行为：

登录信息：
- IP: {ip}
- 账号: {username}
- 时间: {timestamp}
- 失败次数: {fail_count}
- 相同IP历史尝试: {history_count}

请判断：
1. 是否为暴力破解？
2. 是否为异常IP登录？
3. 风险等级（低/中/高）？
4. 建议操作（允许/验证码/阻止）？

返回JSON格式：
{ "risk_level": "low/medium/high", "action": "allow/captcha/block", "reason": "..." }
```

## 4. 评测指标

### 4.1 验证码识别

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 准确率 | ≥90% | 正确识别字符数/总字符数 |
| 召回率 | ≥85% | 成功识别数/总验证码数 |
| 延迟 | <500ms | 单次识别耗时 |
| 误识率 | <3% | 错误识别为其他字符 |

### 4.2 异常检测

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 检出率 | ≥95% | 真实异常行为被检出 |
| 误报率 | <5% | 正常行为被误判为异常 |
| 延迟 | <1s | 分析响应时间 |

## 5. 集成方案

### 5.1 验证码识别流程

```
用户点击验证码 → 前端调用 OCR 服务 → 返回识别结果 → 自动填充
```

**API 接口**：
```
POST /api/ai/captcha-recognize
Body: { "image": "base64..." }
Response: { "result": "ABC123", "confidence": 0.95 }
```

### 5.2 异常检测流程

```
用户提交登录 → 后端调用异常检测 → 返回风险判断 → 执行对应动作
```

**API 接口**：
```
POST /api/ai/login-risk-check
Body: { "ip": "...", "username": "...", "timestamp": "..." }
Response: { "risk_level": "low", "action": "allow" }
```

## 6. 风险与缓解

| 风险 | 等级 | 缓解策略 |
|------|------|----------|
| OCR 识别失败 | 中 | 提供手动输入备选，失败3次后切换验证码 |
| 异常检测误报 | 中 | 新IP默认降低阈值，人工复核通道 |
| API 延迟影响体验 | 低 | 异步处理，前端加 loading 状态 |
| 攻击者绕过 | 高 | 结合行为特征、设备指纹多维度检测 |

## 8. 结论与建议

### 8.1 推荐实施路径

1. **第一阶段（P0）**：验证码识别
   - 优先本地 Tesseract，验证效果后再评估是否升级
   - 目标：提升用户体验，降低输入错误率

2. **第二阶段（P1）**：登录异常检测
   - 基于规则引擎快速上线
   - 积累数据后升级为 AI 模型

3. **第三阶段（P2）**：智能补全
   - 根据用户历史登录数据提供建议

### 8.2 验收条件

- [ ] OCR 识别准确率 ≥ 90%
- [ ] 异常检测误报率 < 5%
- [ ] API 延迟满足 SLA（<1s）
