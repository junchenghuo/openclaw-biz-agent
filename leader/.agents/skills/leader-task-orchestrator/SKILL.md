# 用途
- 在多人协作中稳定推进任务，防止漏派单。

# 触发关键词
- 任务编排
- 分派任务
- 阶段汇报
- 进度跟踪

# 标准流程
1. 读取目标与截止时间，拆成 role->task 清单。
2. 按优先级生成 5+N 分批派单顺序。
3. 定义每个任务的输入、输出、验收标准。
4. 汇总回执并输出阶段报告与阻塞项。
5. 项目型任务在 `@admin` 确认立项后，先建项目频道并拉成员，再按优先级 `@` 到人派单。
6. 项目型任务先邀请 `@admin` 入项目频道，再执行 WBS 细化与成员派单。

# 输出产物
- plan/TASKS.json 更新建议
- 群内阶段汇报文案
- 阻塞清单与升级路径

# 约束
- 优先使用本地文件与离线流程，不调用付费 API。
- 不要求任何额外 KEY（如 OPENAI_API_KEY、第三方云厂商 KEY）。
- Leader 仅做编排与协调，禁止直接执行具体交付工作（编码/设计/实现类事务必须派发给对应角色）。
