# 用途
- 提前暴露风险并给出可执行缓解方案。

# 触发关键词
- 风险台账
- 项目风险
- 风险评估
- 缓解方案

# 标准流程
1. 识别需求、技术、测试、发布四类风险。
2. 按影响度/概率评分并排序。
3. 为 Top 风险生成缓解动作和负责人。
4. 按日更新风险状态与触发条件。

# 输出产物
- RISK_REGISTER.md
- Top 风险处理计划
- 风险变化摘要

# 约束
- 优先使用本地文件与离线流程，不调用付费 API。
- 不要求任何额外 KEY（如 OPENAI_API_KEY、第三方云厂商 KEY）。
