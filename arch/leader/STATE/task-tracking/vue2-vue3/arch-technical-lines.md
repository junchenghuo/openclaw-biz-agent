# Vue2 → Vue3 技术迁移架构方案（初稿）

## 0. 范围与待确认项
### 范围
- 本文聚焦技术分线与风险控制，不覆盖业务优先级排序（由 Product 文档负责）。
- 适用于现有 Vue2 单体/多模块前端系统，目标迁移至 Vue3 稳定可发布形态。

### 待确认项（当前缺失信息）
1. 现网仓库结构（单仓/多仓、子应用边界）。
2. 依赖清单精确版本（vue-router/vuex/UI 库/插件）。
3. 现有构建链路（webpack/vite、babel、ts、eslint、CI）。
4. 灰度发布能力（流量按用户/租户/环境切分能力）。
5. 回滚手段（前端静态资源回滚、网关/BFF 开关、DB 兼容窗口）。

## 3) 可量化门禁指标（Go/No-Go）
| 维度 | 指标 | 门槛 |
|---|---|---|
| 依赖兼容 | 关键依赖兼容通过率 | 100%（不可兼容项需有替代方案并评审通过） |
| 构建稳定性 | 主分支连续构建成功率（7天） | ≥ 98% |
| 测试质量 | 核心链路自动化通过率 | = 100% |
| 缺陷门禁 | P0/P1 未关闭缺陷 | P0=0，P1=0（无缓解不得放行） |
| 权限安全 | 越权/未鉴权访问用例 | 0 个 |
| 性能 | 首屏耗时回归 | 不劣于基线 10%（超过即阻断） |
| 稳定性 | 灰度期 JS Error Rate | 不高于基线 +20% 且绝对值在团队阈值内 |
| 可回滚性 | 回滚演练成功率 | 100%（演练通过后方可正式放量） |

## 4) 触发回滚条件
1. 生产出现 P0 故障（白屏、核心交易链路不可用、鉴权失效）。
2. 关键业务成功率较基线下降 ≥ 5% 且 15 分钟内无法恢复。
3. JS Error Rate 连续 10 分钟超阈值（>基线+20%）并影响核心路径。
4. 性能指标超门限（首屏/交互延迟恶化 >10%）且无快速缓解。
5. 发现越权或安全高危漏洞（未授权访问、敏感信息泄露）。
6. 构建产物异常导致不可恢复发布失败（无法快速热修复）。

**回滚执行要求**
- 通过灰度开关先全量切回 Vue2 稳定版本。
- 同步冻结后续放量，触发故障复盘与根因清单。
- 未完成问题闭环与复演通过前，不得再次放量。
