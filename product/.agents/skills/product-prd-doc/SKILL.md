# 用途
- 快速生成结构化 PRD，并可本地导出文档。

# 触发关键词
- PRD
- 产品需求文档
- .doc
- 需求说明书

# 标准流程
1. 收集目标、用户、范围、验收标准。
2. 生成完整 PRD 章节（背景/流程/功能/边界/验收）。
3. 输出评审版与开发版差异说明。
4. 给出本地导出命令（如 pandoc）与检查项。

# 输出产物
- PRD.md
- PRD 评审清单
- 本地导出 .doc/.docx 说明

# 约束
- 优先使用本地文件与离线流程，不调用付费 API。
- 不要求任何额外 KEY（如 OPENAI_API_KEY、第三方云厂商 KEY）。
