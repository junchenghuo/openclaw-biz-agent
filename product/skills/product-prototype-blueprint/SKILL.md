# 用途
- 在不开付费原型工具的情况下快速定义页面结构。

# 触发关键词
- 原型图
- 线框图
- 页面流程
- 交互草图

# 标准流程
1. 确认页面列表、关键状态和入口出口。
2. 用 Mermaid/ASCII 生成页面流转图。
3. 给出每个页面的组件清单与交互规则。
4. 输出可直接给 UI/前端的原型说明。

# 输出产物
- prototype-flow.md
- 页面线框说明文档
- 组件与交互清单

# 约束
- 优先使用本地文件与离线流程，不调用付费 API。
- 不要求任何额外 KEY（如 OPENAI_API_KEY、第三方云厂商 KEY）。
